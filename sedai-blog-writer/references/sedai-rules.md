# Sedai Blog Rules Reference

## Structure Rules (AEO/GEO/SEO)

- H1 mirrors the exact search query — no colons, no subtitles, no "X: Why It Matters"
- Every H2 is a self-contained block — readable without surrounding context
- First sentence after H2 directly answers the question the heading implies
- Passage length target: 134–167 words per self-contained block
- Evidence density: 3–5 verified statistics per 1,000 words, each with named inline source
- FAQPage: minimum 5 questions, each answer 2–3 sentences, self-contained, schema-ready

## Voice Rules

- Direct — state the point, then support it
- No analogies, ever
- No intro filler ("In today's AI landscape...", "As AI continues to evolve...")
- Short paragraphs — 3–4 sentences maximum
- Transitions must be earned — previous paragraph sets up what comes next
- No "It wasn't about X, it was about Y" constructions
- Em dashes: maximum 3 per post, never in H1 or H2

## Sedai Placement Rules

- Sedai appears once in the body, tied to a specific outcome with a named source
- Post answers the reader's question first — Sedai is one point on a spectrum, not the point of the post
- "Who needs it" sections describe buyer pain in their own language — not job titles, not product benefits
- If a section could only appear on Sedai's website and nowhere else, it is a pitch — rewrite it

## Hard Stops — Never

- No colon in H1 or any H2
- No subtitle pattern in H1
- No em dash in H1 or H2
- No Cast AI references or links (drives traffic to a direct competitor)
- No GigaOm references unless the brief explicitly requires it for a specific contextual reason
- No "visibility" as a Sedai value prop
- No "LLM optimization" as a category label — use "AI agent optimization"
- No Challenge / Approach / Outcome section structure
- No generic CTAs unrelated to the post topic
- Do not use Sedai Style Guide for blog structure — AEO/GEO/SEO research overrides it

## Required Elements Checklist

- [ ] Named author: full name, title, LinkedIn URL
- [ ] Publish date visible
- [ ] Last Updated date visible
- [ ] Meta title matches brief spec
- [ ] Meta description matches brief spec
- [ ] URL slug matches brief spec
- [ ] FAQPage minimum 5 questions
- [ ] All pricing/data tables have datestamped titles
- [ ] Sourcing table output with every claim verified
- [ ] NEEDS SOURCE flags visible at top of draft if any claims are unverified

## Claim Verification Standard

A claim is verified only if:
- It has been fetched from a named primary source (provider pricing page, named study, Sedai internal data, named exec quote)
- The exact number has been confirmed against that primary source
- The source URL is recorded in the sourcing table

A secondary source (another blog, a research summary, a vendor recap) is not a citation. It is a pointer. Follow the pointer to the primary source. If the pointer leads nowhere, the claim does not go in the draft.

Sedai internal sources that count as primary:
- Notes From Suresh
- Sedai webinar data (attributed as "Sedai webinar data, [month year]")
- Sybill call transcripts (VOC)
- Named customer quotes
- Sedai product data / customer outcomes
